Bitacora de desarrollo – Proyecto Deporte Conecta
Fecha: 29 de octubre de 2025
Desarrollador: Sebastian Garcia Cruz
Proyecto: Deporte Conecta – Plataforma colaborativa de conexion deportiva

Avances implementados para esta entrega:

Durante esta fase se consolidaron mejoras clave en la funcionalidad multiusuario, 
la gestion individual de datos y la experiencia de interaccion dentro del mapa. 
Se optimizo el flujo de sesion y se ajustaron detalles visuales para mantener coherencia con el 
nuevo diseño implementado previamente.

Principales mejoras realizadas:

1. Sistema de favoritos individual por usuario
   Se modifico la logica de almacenamiento de favoritos en el mapa, creando una clave unica por 
   usuario (dc_favorites_userId).
   Esto permite que cada usuario tenga su propia lista de lugares marcados sin interferir con los 
   demas perfiles.
   La informacion de favoritos se sincroniza dinamicamente al iniciar sesion y se actualiza en 
   tiempo real dentro del panel lateral.

2. Mejoras en el panel "Cerca de mi"
   Se anadio un boton para seleccionar manualmente el punto central de busqueda en el mapa.
   Se reorganizo la interfaz del panel para que la configuracion de ubicacion y radio de busqueda 
   sea mas clara e intuitiva.
   Se mantiene el soporte para geolocalizacion automatica mediante la API navigator.geolocation.

3. Control de sesion y seguridad visual
   Se fortalecio la validacion de sesion para impedir el acceso directo a map.html sin estar autenticado.
   Se incorporaron controles visuales que identifican correctamente al creador de cada ubicacion.
   Solo el usuario que registro una ubicacion puede eliminarla.

4. Integracion del apartado de perfil
   Se preparo la funcion viewCreatorProfile para dirigir al perfil del usuario actual y dejar la 
   estructura lista para futuros perfiles publicos.
   Se establecio la base para conectar esta vista con un archivo profile.html editable.

5. Mantenimiento de codigo y coherencia general
   Se corrigieron duplicaciones de variables y se refactorizo la gestion de marcadores temporales 
   y circulos del mapa.
   Se mantuvo la compatibilidad entre paneles, modales y configuraciones previas sin romper el flujo 
   actual.

Objetivos en proceso para el siguiente avance – Fase comunidad y perfiles:

En la proxima fase se trabajara en fortalecer el componente social de la aplicacion, centrado en la 
gestion de perfiles y la futura ventana de comunidad. El objetivo es que los usuarios puedan interactuar 
mas entre ellos, compartir intereses y descubrir otros deportistas cercanos.

Objetivo general:
Desarrollar el apartado de comunidad y los perfiles de usuario, permitiendo que cada persona tenga una 
identidad visible dentro de la plataforma y pueda conectar con otros usuarios activos.

Objetivos especificos:

1. Completar la vista de perfil (profile.html) con edicion de datos, avatar, biografia y deportes favoritos.
2. Implementar una vista de perfil publico (user.html) accesible desde el mapa al consultar el creador de una ubicacion.
3. Diseñar la ventana de comunidad con una lista de usuarios activos, buscador y filtros por deporte.
4. Incluir la opcion de seguir a otros usuarios o guardar contactos deportivos.
5. Preparar la estructura para futuras funciones de chat y eventos entre usuarios.

Impacto esperado del siguiente avance:
La integracion del modulo de comunidad y los perfiles personalizados dara a Deporte Conecta una dimension 
social, transformandola en una red deportiva local donde los usuarios no solo comparten ubicaciones sino 
tambien experiencias. Esto sentara la base para la proxima etapa del proyecto, centrada en interaccion y 
colaboracion entre deportistas.

Conclusion:
Con este avance, Deporte Conecta refuerza su identidad como plataforma participativa, incorporando 
elementos de personalizacion y gestion de usuarios. El siguiente paso sera ampliar la funcionalidad 
de perfiles y comunidad, fortaleciendo el sentido de pertenencia y colaboracion dentro de la aplicacion.
